Relational Database [RDBMS]:
    The Database stores data in tables with rows and column,
    enforcing the strict rules (Schemas) to maintain Relationship and data integrity

SQL TYpe Database
    The SQL is used for Querying
    Best for Transactional application where, data  Consistency and ACID properties
    and critical

    ACID [Atomicity , Consistency, Isolation, Durability]

Non SQL Database:
    Build to scale horizontally , NoSql Database handle unstructure or semi structure
    data (Json document,Social media feed) withour requiring predefined Schemas

    High Speed,High Volum,relative application where data structure evolved rapidly



Data Warehouses:

    Specialize Database optimized for analytics rather than daily Transactional processing. 
    They Hold Historical and Integrated data from multiple sources

    EX: Snowflake,Google Big Query,Amazon Redshift

    Best For Business Intelligence (BI),reporting and Complex agregation


Data Lakes:
    Massive storage repository that holds raw,unstructured data in its natural Format untill it is needed

    They Do not Enforce any Schemas upon ingestion,offers high flexibility and scale lower cost.
