AWS[Amazon Web services]:

1.It works pay as you go model
2.Flexible scalling options

1.Regions: Physical locations worldwide where AWS clusters data center
    Example: us-east-1 (Northern part of virgenia)
             ap-south-1(Mumbia)
             
    Optimize latency by chosing regions close to users.


2.Availaibality Zones (AZ):
    Multiple isolated location within each region
    Each AZ contains one or more descrete data centers.
    Ensures High Availaibality(HA) and disaster recovery(DA).
    Prevents single point of failure.



3.Edge location: 
    Smaller data center in major cities worldwide.
    Used to cache content closer to the end user with the help of AWS cloud fund(CDN)
    Reduces the Latency and provide faster content deliver.
